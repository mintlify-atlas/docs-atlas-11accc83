The `tonic::client` module provides the building blocks used by generated client stubs. At its core is the `GrpcService` trait — a thin alias over `tower::Service` — that guarantees the correct request/response body types. Generated clients wrap a `Channel` (or any `GrpcService` implementation) and expose typed methods for each RPC defined in your `.proto` file.

## `GrpcService` trait

`GrpcService<ReqBody>` is the trait every tonic client channel must satisfy. `Channel` from `tonic::transport` implements it automatically.

```rust
pub trait GrpcService<ReqBody> {
    type ResponseBody: http_body::Body;
    type Error: Into<crate::BoxError>;
    type Future: Future<Output = Result<http::Response<Self::ResponseBody>, Self::Error>>;

    fn poll_ready(&mut self, cx: &mut Context<'_>) -> Poll<Result<(), Self::Error>>;
    fn call(&mut self, request: http::Request<ReqBody>) -> Self::Future;
}
```

Any `tower::Service<http::Request<ReqBody>, Response = http::Response<ResBody>>` where `ResBody: http_body::Body` and `ResBody::Error: Into<BoxError>` automatically implements `GrpcService`. You rarely need to implement this trait directly.

### Using generated clients

Code generated by `tonic-build` produces a struct (e.g. `GreeterClient<T>`) that wraps a `T: GrpcService<BoxBody>`. Cloning the client is cheap because the underlying `Channel` is backed by `tower::Buffer`.

```rust
use hello_world::greeter_client::GreeterClient;
use tonic::transport::Channel;

// Create the channel once and clone as needed.
let channel = Channel::from_static("https://[::1]:50051")
    .connect()
    .await?;

let mut client = GreeterClient::new(channel.clone());

let response = client
    .say_hello(tonic::Request::new(HelloRequest {
        name: "Alice".into(),
    }))
    .await?;
```

<Note>
All generated client methods take `&mut self` to satisfy the `tower::Service` contract. Clone the client to send concurrent requests — each clone shares the same connection pool.
</Note>

## `tonic::Request<T>`

`Request<T>` wraps a message `T` together with gRPC metadata (headers) and HTTP extensions. It is used for both outbound client requests and inbound server requests.

```rust
pub struct Request<T> {
    metadata: MetadataMap,
    message: T,
    extensions: Extensions,
}
```

### Constructors

<ParamField path="Request::new" type="fn(message: T) -> Self">
  Creates a new request with an empty `MetadataMap` and no extensions.

  ```rust
  use tonic::Request;

  let req = Request::new(HelloRequest { name: "Bob".into() });
  ```
</ParamField>

<ParamField path="Request::from_parts" type="fn(metadata: MetadataMap, extensions: Extensions, message: T) -> Self">
  Reconstructs a request from its constituent parts. The inverse of `into_parts`.
</ParamField>

### Accessing the message

<ParamField path="get_ref" type="fn(&self) -> &T">
  Returns an immutable reference to the inner message without consuming the request.
</ParamField>

<ParamField path="get_mut" type="fn(&mut self) -> &mut T">
  Returns a mutable reference to the inner message.
</ParamField>

<ParamField path="into_inner" type="fn(self) -> T">
  Consumes the `Request` and returns the inner message, discarding metadata and extensions.
</ParamField>

<ParamField path="into_parts" type="fn(self) -> (MetadataMap, Extensions, T)">
  Consumes the `Request` and returns all three components.
</ParamField>

### Metadata

<ParamField path="metadata" type="fn(&self) -> &MetadataMap">
  Returns an immutable reference to the request metadata (gRPC headers).
</ParamField>

<ParamField path="metadata_mut" type="fn(&mut self) -> &mut MetadataMap">
  Returns a mutable reference to the request metadata. Use this to attach authorization tokens or other headers before sending.

  ```rust
  let mut req = Request::new(HelloRequest { name: "Bob".into() });
  req.metadata_mut().insert(
      "authorization",
      "Bearer my-token".parse().unwrap(),
  );
  ```
</ParamField>

### Extensions

<ParamField path="extensions" type="fn(&self) -> &Extensions">
  Returns an immutable reference to the `http::Extensions` map. Extensions are used to pass arbitrary typed data through the request pipeline.
</ParamField>

<ParamField path="extensions_mut" type="fn(&mut self) -> &mut Extensions">
  Returns a mutable reference to extensions. Interceptors typically insert custom data here for service handlers to read.

  ```rust
  fn intercept(mut req: Request<()>) -> Result<Request<()>, Status> {
      req.extensions_mut().insert(MyExtension { user_id: 42 });
      Ok(req)
  }
  ```
</ParamField>

### Timeout

<ParamField path="set_timeout" type="fn(&mut self, deadline: Duration)">
  Sets the `grpc-timeout` metadata header, informing the server of the maximum time allowed for the RPC. The duration is formatted per the gRPC protocol spec.

  ```rust
  use std::time::Duration;

  let mut req = Request::new(HelloRequest { name: "Bob".into() });
  req.set_timeout(Duration::from_secs(10));
  ```
</ParamField>

## `tonic::Response<T>`

`Response<T>` wraps the server's reply message with trailing metadata and HTTP extensions.

```rust
pub struct Response<T> {
    metadata: MetadataMap,
    message: T,
    extensions: Extensions,
}
```

### Constructors

<ParamField path="Response::new" type="fn(message: T) -> Self">
  Creates a new response wrapping `message` with an empty metadata map. Service implementations use this to return results.

  ```rust
  Ok(Response::new(HelloReply {
      message: format!("Hello, {}!", name),
  }))
  ```
</ParamField>

<ParamField path="Response::from_parts" type="fn(metadata: MetadataMap, message: T, extensions: Extensions) -> Self">
  Constructs a response from its constituent parts.
</ParamField>

### Accessing the message

<ParamField path="get_ref" type="fn(&self) -> &T">
  Borrows the inner message.
</ParamField>

<ParamField path="get_mut" type="fn(&mut self) -> &mut T">
  Mutably borrows the inner message.
</ParamField>

<ParamField path="into_inner" type="fn(self) -> T">
  Consumes the response and returns the message, discarding metadata.
</ParamField>

<ParamField path="into_parts" type="fn(self) -> (MetadataMap, T, Extensions)">
  Consumes the response and returns all three components.
</ParamField>

### Metadata

<ParamField path="metadata" type="fn(&self) -> &MetadataMap">
  Returns the trailing metadata received from the server. Available after the response is fully received.
</ParamField>

<ParamField path="metadata_mut" type="fn(&mut self) -> &mut MetadataMap">
  Mutable access to trailing metadata.
</ParamField>

### Extensions

<ParamField path="extensions" type="fn(&self) -> &Extensions">
  Returns the HTTP extensions attached to the response.
</ParamField>

<ParamField path="extensions_mut" type="fn(&mut self) -> &mut Extensions">
  Mutable access to extensions.
</ParamField>

## `tonic::Streaming<T>`

`Streaming<T>` is the decoded message stream returned by server-streaming and bidirectional-streaming RPC methods. It implements both `Stream<Item = Result<T, Status>>` and provides an async `message()` method for ergonomic iteration.

```rust
pub struct Streaming<T> { /* ... */ }
```

### Methods

<ParamField path="message" type="async fn(&mut self) -> Result<Option<T>, Status>">
  Polls the stream for the next message.

  - `Ok(Some(msg))` — a valid message was received.
  - `Ok(None)` — the stream is closed; no more messages will arrive.
  - `Err(status)` — the sender closed the stream with a gRPC error.

  ```rust
  let mut stream = client.list_features(rect).await?.into_inner();

  while let Some(feature) = stream.message().await? {
      println!("feature: {:?}", feature);
  }
  ```
</ParamField>

<ParamField path="trailers" type="async fn(&mut self) -> Result<Option<MetadataMap>, Status>">
  Drains any remaining messages and returns the trailing metadata sent by the server. If `message()` has already returned `None`, no additional polling is required.

  ```rust
  let trailers = stream.trailers().await?;
  ```
</ParamField>

<Tip>
`Streaming<T>` also implements `tokio_stream::Stream`, so you can use it with `StreamExt` combinators such as `map`, `filter`, and `collect`.
</Tip>
