Tonic is distributed as a set of crates on [crates.io](https://crates.io). The minimum supported Rust version is **1.88**. Before adding tonic to your project, make sure your toolchain is up to date:

```bash
rustup update
```

## Adding dependencies

Add the following to your `Cargo.toml`. The exact combination depends on whether you need a server, a client, or both, but this is the standard starting point for a project that compiles `.proto` files and runs on Tokio:

```toml
[dependencies]
tonic = "0.14"
prost = "0.14"
tonic-prost = "0.14"
tokio = { version = "1.0", features = ["macros", "rt-multi-thread"] }

[build-dependencies]
tonic-prost-build = "0.14"
```

- **`tonic`** — Core gRPC client and server implementation.
- **`prost`** — Protocol Buffers runtime used to serialize and deserialize messages.
- **`tonic-prost`** — Prost codec integration for tonic.
- **`tokio`** — Async runtime. The `macros` feature enables `#[tokio::main]`; `rt-multi-thread` provides the multi-threaded scheduler.
- **`tonic-prost-build`** — Build-time tool that compiles `.proto` files into Rust source code. Goes in `[build-dependencies]` so it is only used during the build step.

### Optional companion crates

Pull in these additional crates as your project grows:

```toml
[dependencies]
tonic-health     = "0.14"  # gRPC health checking service
tonic-reflection = "0.14"  # gRPC server reflection
tonic-types      = "0.14"  # gRPC Well Known Types and utility protos
tonic-web        = "0.14"  # gRPC-Web protocol translation
```

## Feature flags

Tonic's default feature set (`transport`, `router`, `codegen`) is enough for most projects. Individual flags let you trim the dependency tree or opt into TLS and compression.

| Flag | Default | Description |
|---|---|---|
| `transport` | Yes | Full client (`channel`) and server implementation built on hyper, tower, and tokio. Enables both `server` and `channel`. |
| `server` | Via `transport` | Server-only subset of the `transport` feature. |
| `channel` | Via `transport` | Client-channel-only subset of the `transport` feature. |
| `router` | Yes | [axum](https://docs.rs/axum)-based service router for composing multiple gRPC services. |
| `codegen` | Yes | Exports and dependencies required by code generated by `tonic-prost-build`. |
| `tls-ring` | No | TLS via [rustls](https://docs.rs/rustls) using the [ring](https://docs.rs/ring) cryptography provider. |
| `tls-aws-lc` | No | TLS via rustls using the [aws-lc-rs](https://docs.rs/aws-lc-rs) cryptography provider. |
| `tls-native-roots` | No | Adds system trust roots to rustls-based clients via [rustls-native-certs](https://docs.rs/rustls-native-certs). |
| `tls-webpki-roots` | No | Adds the [webpki-roots](https://docs.rs/webpki-roots) certificate bundle to rustls-based clients. |
| `gzip` | No | Gzip compression for requests, responses, and streams. Depends on [flate2](https://docs.rs/flate2). |
| `deflate` | No | Deflate compression for requests, responses, and streams. Depends on [flate2](https://docs.rs/flate2). |
| `zstd` | No | Zstd compression for requests, responses, and streams. Depends on [zstd](https://docs.rs/zstd). |

<Note>
You must pick **at most one** of `tls-ring` and `tls-aws-lc`. Enabling both at the same time is not supported.
</Note>

To enable TLS with system roots, for example:

```toml
[dependencies]
tonic = { version = "0.14", features = ["tls-ring", "tls-native-roots"] }
```

## Installing protoc

`tonic-prost-build` calls the Protocol Buffers compiler (`protoc`) to parse your `.proto` files. You need to install it separately.

<Tabs>
  <Tab title="macOS">
    ```bash
    brew install protobuf
    ```
  </Tab>
  <Tab title="Ubuntu / Debian">
    ```bash
    apt-get install -y protobuf-compiler
    ```
  </Tab>
  <Tab title="Windows">
    Download the latest release from [github.com/protocolbuffers/protobuf/releases](https://github.com/protocolbuffers/protobuf/releases) and add the `bin/` directory to your `PATH`.
  </Tab>
  <Tab title="From source">
    Follow the instructions at [protobuf.dev/downloads](https://protobuf.dev/downloads/).
  </Tab>
</Tabs>

Verify the installation:

```bash
protoc --version
```

<Warning>
Build compilation will fail with a clear error if `protoc` is not on your `PATH` when `tonic-prost-build` runs.
</Warning>

## Setting up build.rs

Create a `build.rs` file at the root of your project (next to `Cargo.toml`) to compile your `.proto` files automatically on each build:

```rust
fn main() -> Result<(), Box<dyn std::error::Error>> {
    tonic_prost_build::compile_protos("proto/helloworld.proto")?;
    Ok(())
}
```

The string argument is the path to your `.proto` file, relative to the project root. The generated Rust code is written to the build output directory and included in your project via the `tonic::include_proto!` macro.

<Tip>
You can call `compile_protos` multiple times to compile several `.proto` files, or explore the `tonic_prost_build::configure()` builder for advanced options such as custom output directories and type attributes.
</Tip>
