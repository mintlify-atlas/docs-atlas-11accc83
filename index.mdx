Tonic is a native Rust implementation of [gRPC](https://grpc.io), a high-performance, open-source RPC framework. It provides everything you need to build production-grade gRPC services: async clients and servers, bi-directional streaming, TLS, interceptors, health checking, and more — all built on the Tokio async runtime.

## Why Tonic?

Tonic is built for Rust developers who need a reliable, ergonomic gRPC library that integrates naturally with the async Rust ecosystem.

## Get started

<Steps>
  <Step title="Add Tonic to your project">
    Add `tonic` and `prost` to your `Cargo.toml`, and `tonic-prost-build` to build dependencies.
  </Step>
  <Step title="Define your service in protobuf">
    Write a `.proto` file defining your gRPC service, request, and response types.
  </Step>
  <Step title="Generate Rust code">
    Use `tonic-prost-build` in your `build.rs` to generate type-safe Rust client and server stubs.
  </Step>
  <Step title="Implement and serve">
    Implement the generated service trait, then start your server with `Server::builder()`.
  </Step>
</Steps>

## Project layout

Tonic is organized into focused crates, each independently usable:

| Crate | Purpose |
|---|---|
| `tonic` | Core gRPC client and server implementation |
| `tonic-build` | Code generation framework and traits |
| `tonic-prost-build` | Prost-based `.proto` compilation |
| `tonic-health` | Standard gRPC health checking service |
| `tonic-reflection` | gRPC server reflection |
| `tonic-types` | gRPC Well-Known Types |
| `tonic-web` | gRPC-Web protocol support |

<Note>
  Tonic's minimum supported Rust version (MSRV) is **1.88**. Run `rustup update` to ensure you have a compatible toolchain.
</Note>
