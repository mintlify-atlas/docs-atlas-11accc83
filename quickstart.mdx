This guide walks you through building a complete gRPC service with Tonic. You will define a `Greeter` service in a `.proto` file, generate typed Rust code from it, implement a server, and call it from a client. The whole thing runs locally on a single machine.

Before starting, make sure you have Rust 1.88 or later and `protoc` installed. See the [installation guide](/installation) if you need to set those up first.

<Steps>
  <Step title="Create a new Rust project">
    Use Cargo to create a new project:

    ```bash
    cargo new helloworld-tonic
    cd helloworld-tonic
    ```
  </Step>

  <Step title="Configure Cargo.toml">
    Replace the contents of `Cargo.toml` with the following. The two `[[bin]]` entries let you run the server and client as separate binaries from the same project.

    ```toml
    [package]
    name = "helloworld-tonic"
    version = "0.1.0"
    edition = "2021"

    [[bin]]
    name = "helloworld-server"
    path = "src/server.rs"

    [[bin]]
    name = "helloworld-client"
    path = "src/client.rs"

    [dependencies]
    tonic = "*"
    prost = "0.14"
    tonic-prost = "*"
    tokio = { version = "1.0", features = ["macros", "rt-multi-thread"] }

    [build-dependencies]
    tonic-prost-build = "*"
    ```
  </Step>

  <Step title="Define the proto file">
    Create the `proto/` directory and a `helloworld.proto` file inside it:

    ```bash
    mkdir proto
    touch proto/helloworld.proto
    ```

    Add the following content to `proto/helloworld.proto`:

    ```proto
    syntax = "proto3";
    package helloworld;

    service Greeter {
        rpc SayHello (HelloRequest) returns (HelloReply);
    }

    message HelloRequest {
        string name = 1;
    }

    message HelloReply {
        string message = 1;
    }
    ```

    The `package` name (`helloworld`) is important — it is the string you pass to `tonic::include_proto!` later.
  </Step>

  <Step title="Set up build.rs for code generation">
    Create a `build.rs` file at the root of your project (next to `Cargo.toml`):

    ```rust
    fn main() -> Result<(), Box<dyn std::error::Error>> {
        tonic_prost_build::compile_protos("proto/helloworld.proto")?;
        Ok(())
    }
    ```

    Cargo runs this file before compiling your crate. It invokes `protoc` on your `.proto` file and writes the generated Rust types and service traits into the build output directory.
  </Step>

  <Step title="Implement the server">
    Create `src/server.rs`:

    ```rust
    use tonic::{transport::Server, Request, Response, Status};

    use hello_world::greeter_server::{Greeter, GreeterServer};
    use hello_world::{HelloReply, HelloRequest};

    pub mod hello_world {
        tonic::include_proto!("helloworld"); // must match the proto package name
    }

    #[derive(Debug, Default)]
    pub struct MyGreeter {}

    #[tonic::async_trait]
    impl Greeter for MyGreeter {
        async fn say_hello(
            &self,
            request: Request<HelloRequest>,
        ) -> Result<Response<HelloReply>, Status> {
            println!("Got a request: {:?}", request);

            let reply = HelloReply {
                message: format!("Hello {}!", request.into_inner().name),
            };

            Ok(Response::new(reply))
        }
    }

    #[tokio::main]
    async fn main() -> Result<(), Box<dyn std::error::Error>> {
        let addr = "[::1]:50051".parse()?;
        let greeter = MyGreeter::default();

        Server::builder()
            .add_service(GreeterServer::new(greeter))
            .serve(addr)
            .await?;

        Ok(())
    }
    ```

    A few things to note:

    - `tonic::include_proto!("helloworld")` brings the generated types into scope. The string must match the `package` name in the `.proto` file.
    - `#[tonic::async_trait]` is required on any `impl` block for a generated service trait, because Rust's trait system does not yet natively support async trait methods.
    - `request.into_inner()` unwraps the `tonic::Request<T>` wrapper to access the inner protobuf message.
  </Step>

  <Step title="Implement the client">
    Create `src/client.rs`:

    ```rust
    use hello_world::greeter_client::GreeterClient;
    use hello_world::HelloRequest;

    pub mod hello_world {
        tonic::include_proto!("helloworld");
    }

    #[tokio::main]
    async fn main() -> Result<(), Box<dyn std::error::Error>> {
        let mut client = GreeterClient::connect("http://[::1]:50051").await?;

        let request = tonic::Request::new(HelloRequest {
            name: "Tonic".into(),
        });

        let response = client.say_hello(request).await?;

        println!("RESPONSE={:?}", response);

        Ok(())
    }
    ```

    The generated `GreeterClient` handles connection management. `connect` takes any URI that resolves to the server address and returns a client ready to make calls.
  </Step>

  <Step title="Run the server and client">
    In one terminal, start the server:

    ```bash
    cargo run --bin helloworld-server
    ```

    In a second terminal, run the client:

    ```bash
    cargo run --bin helloworld-client
    ```

    The server terminal prints the incoming request:

    ```
    Got a request: Request { metadata: MetadataMap { ... }, message: HelloRequest { name: "Tonic" }, ... }
    ```

    The client terminal prints the response:

    ```
    RESPONSE=Response { metadata: MetadataMap { ... }, message: HelloReply { message: "Hello Tonic!" }, ... }
    ```

    You can also test the server with [grpcurl](https://github.com/fullstorydev/grpcurl):

    ```bash
    grpcurl -plaintext -import-path ./proto -proto helloworld.proto \
      -d '{"name": "Tonic"}' '[::1]:50051' helloworld.Greeter/SayHello
    ```

    Expected output:

    ```json
    {
      "message": "Hello Tonic!"
    }
    ```
  </Step>
</Steps>

## What you built

Your project now has the following structure:

```
helloworld-tonic/
├── build.rs
├── Cargo.toml
├── proto/
│   └── helloworld.proto
└── src/
    ├── client.rs
    └── server.rs
```

The `build.rs` script compiles the `.proto` definition into Rust types every time you run `cargo build`. The generated `Greeter` trait is implemented by `MyGreeter` on the server side, and `GreeterClient` is used directly on the client side.
